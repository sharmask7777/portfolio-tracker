# Requirements: Milestone v4.0 - Analytics & Visualization

## 1. Portfolio History & Interactive Graph
*   **Day-to-Day History Engine**: The backend MUST provide an endpoint `/portfolio/history` that returns daily data points for:
    *   `date`: The day of the record.
    *   `investedValue`: Cumulative amount invested up to that date.
    *   `currentValue`: Portfolio value on that date (using historical NAVs).
*   **Interactive Visualization**:
    *   The frontend MUST display an `AreaChart` (or similar) showing both Invested and Current value.
    *   The chart MUST cover the entire duration since the first transaction.
    *   The chart MUST be interactive (hover to see exact values for a date).
    *   The chart SHOULD be navigable/zoomable if the duration is long (e.g., > 5 years).

## 2. Critical Performance Metrics (Insights)
*   **All-Time High (ATH) Tracking**:
    *   System MUST identify the date and value of the highest portfolio corpus ever reached.
    *   System MUST identify the date and value of the maximum amount ever invested.
*   **Yearly Breakdown**:
    *   System MUST provide a breakdown for each calendar year showing:
        *   ATH Corpus Value & Date for that year.
        *   Max Invested Amount & Date for that year.
*   **Display**: These metrics SHOULD be displayed in a dedicated "Historical Insights" section or card.

## 3. UI/UX Refinement & Fixes
*   **Scheme Breakdown Population**:
    *   Investigate and fix why the "Scheme Breakdown" chart appears empty for some users.
    *   Ensure proper labels and tooltips are visible.
*   **Dark Mode Accessibility**:
    *   Fix "black text" issues in dark mode, particularly in `input`, `select`, and certain badge components.
    *   Ensure all text respects the `--text-primary` and `--text-secondary` variables.
*   **Responsive Adjustments**:
    *   Ensure the new history graph is responsive across different screen sizes.
