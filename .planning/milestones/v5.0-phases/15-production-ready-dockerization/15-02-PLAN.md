  <files>frontend/nginx.conf</files>
  <action>
    Create a minimal `nginx.conf` optimized for serving a React SPA:
    - Listen on port 80.
    - Set root to `/usr/share/nginx/html`.
    - Handle SPA routing by redirecting all non-file requests to `index.html` (`try_files $uri /index.html`).
    - Enable basic gzip compression.
    - Set appropriate cache headers for static assets (optional but recommended).
  </action>
  <verify>
    grep "try_files \$uri /index.html" frontend/nginx.conf
  </verify>
  <done>Nginx configuration created for SPA support.</done>
</task>

<task type="auto">
  <name>Task 3: Create multi-stage Dockerfile for Frontend</name>
  <files>frontend/Dockerfile</files>
  <action>
    Create a multi-stage `Dockerfile` in the `frontend/` directory:
    
    Stage 1: Build
    - Use `node:20-alpine`.
    - Copy package files and install all dependencies (`npm ci`).
    - Copy source code.
    - Run `npm run build`.
    
    Stage 2: Runtime
    - Use `nginx:stable-alpine`.
    - Copy the custom `nginx.conf` to `/etc/nginx/conf.d/default.conf`.
    - Copy the built assets from Stage 1 (`dist/`) to `/usr/share/nginx/html`.
    - EXPOSE 80.
    - CMD ["nginx", "-g", "daemon off;"].
  </action>
  <verify>
    docker build -t portfolio-tracker-frontend ./frontend
  </verify>
  <done>Frontend Dockerfile created and image builds successfully.</done>
</task>
