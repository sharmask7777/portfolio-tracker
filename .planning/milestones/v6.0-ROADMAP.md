# Project Roadmap

## Milestone v6.0: Authentication & Multi-User Support

## Phase 18: Backend Authentication Core [DONE]
**Goal:** Implement the foundation for secure authentication and user management.

**Success Criteria:**
1. Users can register and login via API.
2. Passwords are encrypted in the database.
3. JWT tokens are issued upon successful login.
4. `authMiddleware` correctly identifies users from tokens and rejects unauthorized requests.

## Phase 19: Frontend Authentication & Identity [DONE]
**Goal:** Connect the frontend to the authentication system.

**Success Criteria:**
1. Login and Signup pages are functional and visual.
2. `AuthContext` successfully persists the user session (JWT).
3. Private routes (Dashboard, Family View, etc.) are only accessible to logged-in users.
4. Logout clears the session and redirects to the login page.

## Phase 20: Multi-User Data Isolation & Refactoring [DONE]
**Goal:** Ensure every user's data is strictly isolated and refactor existing routes.

**Success Criteria:**
1. All API routes use the authenticated user's ID.
2. Data isolation is verified through tests (one user cannot see another's data).
3. User profile information is correctly displayed and editable.
4. System no longer relies on `mock-user-123`.
