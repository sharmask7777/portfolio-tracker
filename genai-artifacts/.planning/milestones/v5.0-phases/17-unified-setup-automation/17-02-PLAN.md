       - Move `prisma` from `devDependencies` to `dependencies`.

       - In Stage 2 (deps), ensure `prisma` is installed by using `npm ci --omit=dev` after moving it to `dependencies`.
       - In Stage 3 (Final Runtime):
         - Install `openssl` and `libc6-compat` (required by Prisma).
         - Copy `prisma/migrations` folder: `COPY --from=builder /app/prisma/migrations ./prisma/migrations`.

  </action>

<task type="auto">
  <name>Task 2: Implement one-command start orchestration</name>
  <files>package.json, setup.sh</files>
  <action>
    1. Update root `package.json`:
       - Add `docker:up`: `docker-compose up --build -d`
       - Add `docker:down`: `docker-compose down`
       - Add `docker:logs`: `docker-compose logs -f`
       - Add `start`: `npm run docker:up && npm run docker:logs`
    2. Update `setup.sh`:
       - Add a prompt at the end: "Setup complete. Would you like to start the application via Docker? (y/n)".
       - If 'y', execute `docker-compose up --build -d`.
       - Inform the user how to see logs or access the dashboard.
  </action>
  <verify>
    <automated>
      npm run docker:up -- --help && grep "docker-compose up" setup.sh
    </automated>
  </verify>
  <done>The application can be started with a single command from setup.sh or root package.json.</done>
</task>
